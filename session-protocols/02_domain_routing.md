# domain routing table

when Claude loads files at session start, it should load only
what's relevant to the current task — not everything in the vault.
this table maps tasks to the files that actually matter for them.

adapt the file paths to match your own vault structure.

---

## routing table

### ✍️ content (posts, threads, newsletter, scheduling)

**always load:**
- content manual
- working manual (what's active, what to avoid)
- content pillars

**load additionally if:**
- writing AI/Claude content → AI operating system file
- writing identity or psychology posts → identity analysis file
- writing music content → music manual
- visual or art post → visual brand spec
- service CTA needed → services file
- newsletter specifically → newsletter manual

---

### 🎵 music (lyrics, releases, Suno, distribution)

**always load:**
- music manual
- discography / release catalog

**load additionally if:**
- writing lyrics → lyric analysis file + 3 existing lyric files
- Suno prompt work → Suno analysis file
- building music content → also load content manual
- distribution or admin → all links file

---

### 🤖 AI workflows (building, documenting, experimenting)

**always load:**
- AI operating system file
- workflow system file

**load additionally if:**
- turning workflow into content → also load content manual
- adding a new tool → update the AI operating system after session

---

### 💼 business (services, consulting, products)

**always load:**
- services ecosystem file
- business README

**load additionally if:**
- ad campaigns → ad strategy file
- social proof needed → testimonials file
- pricing or CTA → services file has everything
- turning business content into posts → also load content manual

---

### 🧠 identity / voice calibration

**always load:**
- who am I file
- cognitive profile / tweet profile

**load additionally if:**
- writing identity posts → identity posts analysis
- psychology pillar content → emotional architecture file
- deep voice calibration → voice deep-dive file

---

## the rule

load files for the domain. not the whole vault.
if the task expands mid-session and new files are needed,
load them then — not upfront as a precaution.

modular reads keep sessions fast and focused.
